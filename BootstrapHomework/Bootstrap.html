Alert

Über Alerts

Mit einem Alert können Sie kontextbezogene Rückmeldungen für typische Benutzeraktionen bereitstellen. Hierfür gibt es vordefinierte Meldungen wie z.B. "Warning" oder "Info" allerdings sind Meldungen auch für jede Textlänge sowie eine optionale Schaltfläche zum Verwerfen verfügbar.

alert.class

Für Alerts stehen vordefinierte Klassen zur Verfügung:

  • alert-success
  • alert-info
  • alert-warning
  • alert-danger
  • alert-primary
  • alert-secondary
  • alert-dark
  • alert-light


alert-link

Mit dem alert-link ist es möglich, farblich passende links innerhalb eines Alerts zu platzieren.



Erweiteter Inhalt

Es ist durchaus möglich, dass ein Alert andere HTML Elemente enthält wie zum Beispiel Absätze oder Überschriften.




Card

Über Cards

Eine Card ist ein flexibler und erweiterbarer Inhaltscontainer. Sie enthält Optionen für Kopf- und Fußzeilen, eine Vielzahl von Inhalten, kontextbezogene Hintergrundfarben und leistungsstarke Anzeigeoptionen. Cards werden mit so wenig Aufschlag und Stilen wie möglich erstellt, bieten aber dennoch eine Menge Kontrolle und Anpassungsmöglichkeiten. Sie sind mit einer Flexbox ausgestattet und lassen sich leicht ausrichten und gut mit anderen Bootstrap-Komponenten kombinieren. Sie haben standardmäßig keinen Rand. Verwenden Sie daher nach Bedarf die Abstandshilfsprogramme. Cards haben zu Beginn keine feste Breite, daher füllen sie natürlich die gesamte Breite des übergeordneten Elements aus. Dies lässt sich mit den verschiedenen Größenoptionen leicht anpassen.

Inhalte

Für Cards gibt es verschiedene Möglichkeiten für Inhalte:

  • .card-body: Ist sozusagen die Inhaltsbox einer Card. Wird vor allem verwendet, wenn man ein Padding braucht.
  • .card-title: Hierdurch können der Card Titel hinzugefügt werden.
  • .card-subtitle: Für Untertitel.
  • .card-link: Um links in der Card unterzubringen.
  • .card-text: Für normale Textelement in der Card.
  • .card-img-top: Um ein Bild zum Beipiel oben in einer Card einzufügen.

    • Des weiteren kann man Cards noch list groups hinzufügen.

    Über Carousel

    Das Carousel ist eine Diashow zum Durchlaufen einer Reihe von Inhalten, die mit CSS-3D-Transformationen und etwas JavaScript erstellt wurden. Es funktioniert mit einer Reihe von Bildern, Texten oder benutzerdefinierten Markups. Es enthält auch Unterstützung für vorherige / nächste Steuerelemente und Anzeigen. In Browsern, in denen das Page Visibility API unterstützt wird, verhindert das Karussell das Verrutschen, wenn die Webseite für den Benutzer nicht sichtbar ist (z. B. wenn die Registerkarte des Browsers inaktiv ist, das Browserfenster minimiert ist usw.). Bitte beachten Sie, dass verschachtelte Carousel nicht unterstützt werden und Carousel im Allgemeinen nicht den Barrierefreiheitsstandards entsprechen. Carousel normalisieren die Objektträgerabmessungen nicht automatisch. Daher müssen Sie möglicherweise benutzerdefinierte Stile verwenden, um die Größe des Inhalts entsprechend anzupassen. Carousel unterstützen zwar die vorherigen / nächsten Bedienelemente und Anzeigen, sind jedoch nicht ausdrücklich erforderlich. Stellen Sie sicher, dass Sie eine eindeutige ID für optionale Steuerelemente im Karussell festlegen, insbesondere, wenn Sie mehrere Karusselle auf einer Seite verwenden.

    Dropdown

    Über Dropdowns

    Dropdowns sind umschaltbare, kontextbezogene Overlays zum Anzeigen von Linklisten und mehr. Sie werden mit dem im Lieferumfang enthaltenen JavaScript-Dropdown-Plugin "Bootstrap" interaktiv gemacht. Sie werden durch Klicken und nicht durch Schweben umgeschaltet. Dies ist eine beabsichtigte Entwurfsentscheidung.Dropdowns basieren auf einer Bibliothek von Drittanbietern, Popper.js, die eine dynamische Positionierung und Erkennung von Ansichtsfenstern bietet. Stellen Sie sicher, dass Sie popper.min.js vor dem JavaScript von Bootstrap einschließen, oder verwenden Sie bootstrap.bundle.min.js / bootstrap.bundle.js, das Popper.js enthält. Popper.js wird jedoch nicht zum Positionieren von Dropdowns in Navigationsleisten verwendet, da keine dynamische Positionierung erforderlich ist.

    Einzelne Dropdown Schaltfläche

    Jede einzelne .btn-Datei kann mit einigen Markup-Änderungen in ein Dropdown-Menü umgewandelt werden. So können Sie sie für beide Button-Elemente verwenden:




    Dropdown mit geteiltem Button

    In ähnlicher Weise können Sie Dropdown-Listen mit geteilten Schaltflächen mit praktisch demselben Markup wie Dropdown-Listen mit einzelnen Schaltflächen erstellen, jedoch mit dem Zusatz .dropdown-toggle-split, um den richtigen Abstand um das Dropdown-Caret zu gewährleisten.Wir verwenden diese zusätzliche Klasse, um die horizontale Polsterung auf beiden Seiten des Einfügemarke um 25% zu reduzieren und den linken Rand zu entfernen, der für reguläre Dropdown-Schaltflächen hinzugefügt wurde. Durch diese zusätzlichen Änderungen bleibt das Einfügemarke in der geteilten Schaltfläche zentriert, und neben der Hauptschaltfläche wird ein Trefferbereich mit einer angemesseneren Größe angezeigt.






    Navbar

    Über Navbars

    Dokumentation und Beispiele für den leistungsstarken, reaktionsschnellen Navigations-Header von Bootstrap, die Navigationsleiste. Beinhaltet Unterstützung für Branding, Navigation und mehr, einschließlich Unterstützung für unser Collapse-Plugin.

    Wie es funktioniert

    Das brauchen Sie für eine funktionierende Navbar:

    • Für Navigationsleisten ist ein Umbruch von .navbar mit .navbar-expand {-sm | -md | -lg | -xl} erforderlich, damit die Klassen für auf das Reduzieren des Fenstert und das Farbschema reagieren können.
    • Navigationsleisten und deren Inhalt sind standardmäßig flüssig. Verwenden Sie optionale Behälter, um die horizontale Breite zu begrenzen.
    • Verwenden Sie unsere Dienstprogrammklassen "Abstand" und "Flex", um den Abstand und die Ausrichtung innerhalb von Navigationsleisten zu steuern.
    • Navigationsleisten reagieren standardmäßig, Sie können sie jedoch problemlos ändern, um dies zu ändern. Das Reaktionsverhalten hängt von unserem Collapse JavaScript-Plugin ab.
    • Navigationsleisten werden beim Drucken standardmäßig ausgeblendet. Erzwingen Sie das Drucken der Dateien, indem Sie der .navbar .d-print hinzufügen. Siehe die Anzeige-Dienstprogrammklasse.
    • Stellen Sie die Barrierefreiheit sicher, indem Sie ein .nav-Element verwenden, oder fügen Sie bei Verwendung eines allgemeineren Elements wie .div jeder Navigationsleiste eine role = "navigation" hinzu, um sie explizit als Orientierungspunkt für Benutzer von Hilfstechnologien zu kennzeichnen.


    Unterstützte Elemente:

    • .navbar-brand: für Name von Produkten.
    • .navbar-nav: für eine Navigation mit voller Höhe und geringem Gewicht (einschließlich Unterstützung für Dropdowns).
    • .navbar-toggler: zur Verwendung mit unserem Einklapp-Plugin und anderen Verhaltensweisen zum Umschalten der Navigation.
    • .form-inline: für alle Formularsteuerelemente und Aktionen.
    • .navbar-text: zum Hinzufügen vertikal zentrierter Textzeichenfolgen.
    • .collapse.navbar-collapse: zum Gruppieren und Ausblenden von Navigationsleisteninhalten nach einem übergeordneten Haltepunkt.

    Navbar Brand und Bilder

    Die .navbar-brand kann auf die meisten Elemente angewendet werden, ein Anker funktioniert jedoch am besten, da für einige Elemente möglicherweise Dienstprogrammklassen oder benutzerdefinierte Stile erforderlich sind. Das Hinzufügen von Bildern zur .navbar-brand erfordert wahrscheinlich immer benutzerdefinierte Stile oder Hilfsprogramme, um die richtige Größe zu erhalten.     List Group

    Über List Groups

    Listengruppen sind eine flexible und leistungsstarke Komponente zum Anzeigen einer Reihe von Inhalten. Ändern und erweitern Sie sie, um so gut wie jeden Inhalt zu unterstützen.

    Basics



    Aktive List Group Elemente

    Füge .active zu einem .list-Gruppenelement hinzu, um die aktuell aktive Auswahl anzuzeigen.

    Deaktivierte List Group Elemente

    Füge .disabled zu einem .list-group-Element hinzu, damit es deaktiviert erscheint. Beachten Sie, dass für einige Elemente mit der Erweiterung .disabled auch benutzerdefiniertes JavaScript erforderlich ist, um ihre Klickereignisse (z. B. Links) vollständig zu deaktivieren.


    Links und Buttons in List Groups

    Verwenden Sie .a s oder .button's, um aktivierbare Listengruppenelemente mit Hover-, Deaktivierungs- und Aktivierungsstatus zu erstellen, indem Sie die Aktion .list-group-item hinzufügen. Wir trennen diese Pseudoklassen, um sicherzustellen, dass Listengruppen, die aus nicht interaktiven Elementen (wie .li's oder .div's) bestehen, keine Klick- oder Tap-Vorteile bieten.


    Flush

    Fügen Sie .list-group-flush hinzu, um einige Ränder und abgerundete Ecken zu entfernen und Listengruppenelemente in einem übergeordneten Container (z. B. Karten) randlos darzustellen.


    Kontextbezogene Klassen

    Verwenden Sie Kontextklassen, um Listenelemente mit einem statusbezogenen Hintergrund und einer statusbezogenen Farbe zu formatieren.


    Modal

    Über Modals

    Verwenden Sie das modale JavaScript-Plugin von Bootstrap, um Ihrer Site Dialogfelder für Leuchtkästen, Benutzerbenachrichtigungen oder vollständig benutzerdefinierten Inhalt hinzuzufügen.

    Wie sie funktionieren

    Das brauchen Sie für eine funktionierende Navbar:

    • Modals werden mit HTML, CSS und JavaScript erstellt. Sie befinden sich über allen anderen Elementen im Dokument und entfernen den Bildlauf vom .body, sodass der modale Inhalt stattdessen einen Bildlauf durchführt.
    • Durch Klicken auf den modalen „Hintergrund“ wird der modale automatisch geschlossen.
    • Bootstrap unterstützt jeweils nur ein modales Fenster. Geschachtelte Modals werden nicht unterstützt, da man der Meinung ist, dass sie eine schlechte Benutzererfahrung darstellen.
    • Modals verwenden Position: Fixed, was manchmal ein bisschen spezifisch für das Rendering sein kann. Platzieren Sie Ihren modalen HTML-Code nach Möglichkeit an einer erstklassigen Position, um mögliche Interferenzen durch andere Elemente zu vermeiden. Beim Verschachteln eines .modal-Objekts in ein anderes festes Element treten wahrscheinlich Probleme auf.
    • Auch hier gibt es aufgrund der Position: behoben einige Einschränkungen bei der Verwendung von Modals auf Mobilgeräten. Weitere Informationen finden Sie in unseren Dokumenten zur Browserunterstützung.


    Komponenten

    Unten sehen Sie ein statisches modales Beispiel (dh Position und Anzeige wurden überschrieben). Enthalten sind der modale Header, der modale Body (zum Auffüllen erforderlich) und der modale Footer (optional). Wir bitten Sie, modale Header mit Entlassungsaktionen einzuschließen, wann immer dies möglich ist, oder eine andere explizite Entlassungsaktion bereitzustellen.


    Positionierung und Scrollen

    Scrolling: Wenn die Inhalte für das Ansichtsfenster oder das Gerät des Benutzers zu lang werden, werden sie unabhängig von der Seite selbst verschoben. Hinsichtlich der Positionierung: Fügen Sie .modal-dialog-centered zu .modal-dialog hinzu, um das Modal vertikal zu zentrieren.


    Tooltips

    Tooltips und Popovers können nach Bedarf in Modals platziert werden. Wenn Modals geschlossen werden, werden alle darin enthaltenen QuickInfos und Popovers ebenfalls automatisch geschlossen.


    Pagination

    Über Pagination

    Wir verwenden einen großen Block verbundener Links für unsere Paginierung, sodass Links nur schwer zu übersehen sind und problemlos skaliert werden können - und dies bei großen Trefferflächen. Die Paginierung wird mit Listen-HTML-Elementen erstellt, sodass Screenreader die Anzahl der verfügbaren Links anzeigen können. Verwenden Sie ein umschließendes .nav -Element, um es als Navigationsabschnitt für Screenreader und andere unterstützende Technologien zu kennzeichnen. Da Seiten wahrscheinlich mehr als einen solchen Navigationsabschnitt haben, ist es ratsam, eine beschreibende Arienbezeichnung für das .nav bereitzustellen, um dessen Zweck widerzuspiegeln. Wenn zum Beispiel die Paginierungskomponente zum Navigieren zwischen einer Reihe von Suchergebnissen verwendet wird, könnte eine geeignete Bezeichnung aria-label = "Suchergebnisseiten" sein.

    Icons

    Möchten Sie ein Symbol oder ein Symbol anstelle von Text für einige Paginierungslinks verwenden? Stellen Sie sicher, dass der Screenreader mit den Attributen aria und dem Dienstprogramm .sr only ordnungsgemäß unterstützt wird.


    Aktivieren und Deaktivieren

    Paginierungslinks können für verschiedene Umstände angepasst werden. Verwenden Sie .disabled für Links, die als nicht klickbar und .active zur Anzeige der aktuellen Seite angezeigt werden.Während die .disabled-Klasse Zeigerereignisse verwendet: none, um die Linkfunktionalität von .a's zu deaktivieren, ist diese CSS-Eigenschaft noch nicht standardisiert und berücksichtigt die Tastaturnavigation nicht. Aus diesem Grund sollten Sie deaktivierten Links immer tabindex = "- 1" hinzufügen und benutzerdefiniertes JavaScript verwenden, um deren Funktionalität vollständig zu deaktivieren.

    Toasts

    Über Toasts

    Push-Benachrichtigungen an Ihre Besucher mit einem Toast, einer einfachen und leicht anpassbaren Warnmeldung. Toasts sind einfache Benachrichtigungen, die die von Mobil- und Desktop-Betriebssystemen verbreiteten Push-Benachrichtigungen imitieren sollen. Sie sind mit einer Flexbox ausgestattet und daher einfach auszurichten und zu positionieren.

    Basics

    Um erweiterbare und vorhersehbare Toasts zu fördern, empfehlen wir einen Header und einen Body. Toast-Header verwenden display: flex und ermöglichen dank unserer Dienstprogramme margin und flexbox die einfache Ausrichtung von Inhalten. Toasts sind so flexibel, wie Sie es brauchen, und erfordern nur sehr wenig Aufschlag. Zumindest benötigen wir ein einzelnes Element, um Ihren "gerösteten" Inhalt zu enthalten, und fordern Sie nachdrücklich zum Schließen auf.



    Durchscheinend

    Toasts sind ebenfalls leicht durchscheinend, sodass sie über alles, worüber sie erscheinen mögen, übergehen. Bei Browsern, die die Hintergrundfilter-CSS-Eigenschaft unterstützen, wird auch versucht, die Elemente unter einem Toast zu verwischen.

    Stacking

    Wenn Sie mehrere Toasts haben, stapeln sich diese automatisch standardmäßig vertikal in lesbarer Weise.

    Positionierung

    Platzieren Sie Toasts mit benutzerdefiniertem CSS nach Bedarf. Das obere rechte Feld wird häufig für Benachrichtigungen verwendet, ebenso wie das obere mittlere Feld. Wenn Sie immer nur einen Toast gleichzeitig anzeigen möchten, platzieren Sie die Positionierungsstile direkt auf dem Toast.
